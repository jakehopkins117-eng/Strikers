STRIKERS v3.1.1 — PREDICTION INTEGRITY

What changed
- Scheduled-game predictions now send MLB gamePk and official_date from the React frontend.
- Re-running the same MLB game updates its existing SQLite record instead of adding a new record.
- Dashboard/history totals now use one SQLite row per game.
- Existing repeated legacy checks for the same matchup on the same calendar date are collapsed to the newest record when the database initializes.
- The legacy JSON history is also updated in place for repeated scheduled-game predictions.
- Manual matchup predictions still work. Scheduled games provide the strongest duplicate protection because they include MLB gamePk.

Files changed
- web_api.py
- services/prediction_database.py
- frontend/src/App.tsx
